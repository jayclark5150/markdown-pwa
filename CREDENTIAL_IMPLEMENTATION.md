# Environment-Based Credential Implementation

## Overview

After rotating credentials, we'll implement a secure system to manage them without hardcoding in config.js. This prevents credentials from being exposed in git history or browser source code.

---

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│ Build/Deploy Process                                    │
├─────────────────────────────────────────────────────────┤
│ 1. GitHub Actions reads secrets from Secrets store      │
│ 2. Build script injects credentials into HTML/JS        │
│ 3. Deployed PWA has credentials loaded at runtime       │
│ 4. Source code never contains actual credentials        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ Local Development                                       │
├─────────────────────────────────────────────────────────┤
│ 1. Developer creates local config.js (gitignored)       │
│ 2. App reads from config.js                             │
│ 3. Local testing works with real credentials            │
│ 4. config.js never committed to git                     │
└─────────────────────────────────────────────────────────┘
```

## File Changes

### 1. Create `config.example.js` (Committed to Git)

**Purpose:** Shows developers how to set up credentials locally

```javascript
// ⚠️ DO NOT EDIT THIS FILE
// This file shows the expected structure for credentials.
// 
// For LOCAL DEVELOPMENT:
// 1. Copy this file to config.js
// 2. Replace PLACEHOLDER_* with your actual credentials
// 3. Never commit config.js to git (it's in .gitignore)
//
// For DEPLOYMENT:
// Credentials are injected by the build process from GitHub Secrets.
// See CREDENTIAL_ROTATION_GUIDE.md for setup instructions.

window.APP_CONFIG = {
  GOOGLE_CLIENT_ID: 'PLACEHOLDER_CLIENT_ID',
  GOOGLE_API_KEY:   'PLACEHOLDER_API_KEY',
};
```

### 2. Update `app.js` (Credentials Loading)

Replace the hardcoded config loading with robust fallback:

```javascript
// ── Config ────────────────────────────────────────────────────────────────────
// Load credentials from:
// 1. window.APP_CONFIG (set by config.js for local dev, or injected by build)
// 2. Environment (for server-side rendering if applicable)
// 3. Graceful fallback with helpful error

const APP_CONFIG = window.APP_CONFIG || {};
const GOOGLE_CLIENT_ID = APP_CONFIG.GOOGLE_CLIENT_ID || '';
const GOOGLE_API_KEY   = APP_CONFIG.GOOGLE_API_KEY   || '';

// Warn if credentials are missing in development
if (typeof window !== 'undefined' && window.location.hostname === 'localhost') {
  if (!GOOGLE_CLIENT_ID || !GOOGLE_API_KEY) {
    console.warn(
      'Google credentials not found. For local development:\n' +
      '1. Copy config.example.js to config.js\n' +
      '2. Replace PLACEHOLDER_* with actual credentials\n' +
      'See CREDENTIAL_ROTATION_GUIDE.md for setup.'
    );
  }
}

const SCOPES = 'https://www.googleapis.com/auth/drive.file';
const DISCOVERY_DOC = 'https://www.googleapis.com/discovery/v1/apis/drive/v3/rest';
```

### 3. Update `.github/workflows/deploy.yml` (CI/CD)

**Before (INSECURE):**
```yaml
env:
  GOOGLE_CLIENT_ID: '505353054413-...' # ❌ EXPOSED IN LOGS
  GOOGLE_API_KEY: 'AIzaSyCO0KsH29J0MPD9...' # ❌ EXPOSED IN LOGS
```

**After (SECURE):**
```yaml
env:
  GOOGLE_CLIENT_ID: ${{ secrets.GOOGLE_CLIENT_ID }} # ✅ MASKED IN LOGS
  GOOGLE_API_KEY: ${{ secrets.GOOGLE_API_KEY }} # ✅ MASKED IN LOGS

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Inject credentials
        run: |
          cat > config.js << EOF
          window.APP_CONFIG = {
            GOOGLE_CLIENT_ID: '${{ secrets.GOOGLE_CLIENT_ID }}',
            GOOGLE_API_KEY: '${{ secrets.GOOGLE_API_KEY }}',
          };
          EOF
        shell: bash
      
      # ... rest of deploy steps
      # config.js with real credentials is now available
      # but never committed to git
```

### 4. Update `.gitignore` (Already Done, Verify)

Make sure these lines are present:

```
# Credentials
config.js
.env
.env.local
.env.*.local

# IDE
.vscode/
.idea/
*.swp
```

## Implementation Steps

### Step 1: Create `config.example.js`

```bash
cd /Users/jayclark/Documents/markdown-pwa-main
cat > config.example.js << 'EOF'
// ⚠️ DO NOT EDIT THIS FILE
// This file shows the expected structure for credentials.
// 
// For LOCAL DEVELOPMENT:
// 1. Copy this file to config.js
// 2. Replace PLACEHOLDER_* with your actual credentials
// 3. Never commit config.js to git (it's in .gitignore)
//
// For DEPLOYMENT:
// Credentials are injected by the build process from GitHub Secrets.
// See CREDENTIAL_ROTATION_GUIDE.md for setup instructions.

window.APP_CONFIG = {
  GOOGLE_CLIENT_ID: 'PLACEHOLDER_CLIENT_ID',
  GOOGLE_API_KEY:   'PLACEHOLDER_API_KEY',
};
EOF
```

### Step 2: Remove `config.js` from Git History

```bash
# Remove config.js from git history permanently
git rm --cached config.js
git commit -m "chore: remove credentials from git history"

# Verify it's not tracked
git ls-files | grep config.js  # Should show nothing
```

### Step 3: Update `.gitignore`

Make sure `config.js` is listed:

```bash
echo "config.js" >> .gitignore
git add .gitignore
git commit -m "chore: ensure config.js is gitignored"
```

### Step 4: Update `app.js`

Replace the credentials loading section (top of file) with the code shown above.

### Step 5: Update GitHub Actions Workflow

Update `.github/workflows/deploy.yml` to inject credentials at build time (see example above).

### Step 6: Commit and Push

```bash
git add config.example.js app.js .github/workflows/deploy.yml .gitignore
git commit -m "refactor: implement environment-based credential management"
git push origin main
```

## Local Development Setup

After the code changes are live:

1. **Create local config.js:**
   ```bash
   cp config.example.js config.js
   ```

2. **Edit config.js with your new credentials:**
   ```bash
   # Replace PLACEHOLDER_* with actual values from Step 2 of rotation guide
   # GOOGLE_CLIENT_ID: paste from Google Cloud Console
   # GOOGLE_API_KEY: paste from Google Cloud Console
   ```

3. **Test locally:**
   - Open http://localhost:8000
   - Sign in to Google Drive
   - Verify all Drive features work

4. **Never commit config.js:**
   - It's in .gitignore
   - Running `git status` should not show config.js
   - Running `git ls-files` should not show config.js

## Deployment Verification

After deploying with the updated workflow:

1. **Verify credentials are injected:**
   - Deploy the PWA
   - Open in browser and check:
     - Sign in works
     - Can open files from Drive
     - Can save to Drive
     - Can rename/delete files

2. **Verify no secrets in logs:**
   - Go to GitHub Actions > workflow run
   - Check build logs for your credentials
   - Should see: `GOOGLE_CLIENT_ID: ***` (masked)
   - Should not see actual Client ID or API Key

3. **Verify no secrets in git:**
   - Run: `git log --all -p -- config.js | head -50`
   - Should show removal commit
   - Should not show any actual credentials in history

## Rollback Plan (If Something Goes Wrong)

If credentials are compromised again:

1. **Immediately rotate again** in Google Cloud Console (delete and recreate)
2. **Update GitHub Secrets** with new values
3. **Redeploy** the application
4. **Monitor API usage** for suspicious activity
5. **Check git logs** for any leaks: `git log --all -p --source -- config.js`

---

## Benefits

✅ **No credentials in git history** — Can share repo publicly  
✅ **No credentials in source code** — Safe to review code  
✅ **No credentials in browser console** — Even if XSS happens, attacker can't find them  
✅ **Environment isolation** — Different credentials for dev/staging/prod  
✅ **Easy rotation** — Only need to update GitHub Secrets  
✅ **Secure CI/CD** — Credentials masked in build logs
